## Overview

**AuralOf** is a full-stack audio analysis platform that identifies whether music is **Human-Made** or **AI-Generated** using advanced deep learning and audio fingerprinting technologies. Beyond classification, it provides intelligent song recommendations by finding the most similar real-world tracks to any uploaded audio.

At its core, AuralOf combines:

- **Deep Learning Classification**: A custom CNN model trained on Mel spectrograms to distinguish human compositions from AI-generated music
- **Similarity Search Engine**: Advanced audio embeddings combined with FAISS technology to find the top $k$ most similar human-made tracks
- **Privacy-Focused Processing**: Automatic deletion of human-made tracks while securely analyzing AI-generated content

---

## Understanding the Two Audio Types

### 🎼 **Human-Generated Sound**

When you upload audio created by real musicians, producers, or bands:

- **What Happens**: The system analyzes the sound characteristics and determines it's human-made
- **Result**: You're notified that the track is authentic human creation
- **Privacy**: The file is **immediately deleted** after analysis—we don't store human-created music
- **What You Learn**: Confirmation that the music was created by human artists
- **Use Case**: Verify original compositions, protect artist recognition, or ensure authentic content

### 🤖 **AI-Generated Sound**

When you upload audio created by artificial intelligence (music generation models):

- **What Happens**: The system identifies the track as AI-generated and analyzes its audio characteristics
- **Result**: You receive a detailed result showing the AI detection confidence
- **Similarity Matching**: The system finds the **top-k most similar human-made songs** that match the style, genre, and musical characteristics of your AI upload
- **Recommendations**: Get curated suggestions for real songs that sound similar to the AI-generated track
- **Use Case**: Discover authentic music similar to AI compositions, understand musical influences, or find inspiration for artist-created alternatives

## Key Features

✨ **AI vs. Human Detection**

- Custom Convolutional Neural Network trained on Mel spectrograms
- Classifies audio files with high accuracy (human-made vs. AI-generated)
- Real-time analysis with instant results

🔍 **Smart Similarity Search**

- For AI-generated tracks: automatically finds the top $k$ most similar human-made songs
- Uses **TorchOpenL3** for advanced audio embeddings
- Powered by **FAISS** (Facebook AI Similarity Search) for lightning-fast similarity matching
- Considers waveform embeddings and genre similarity for accurate recommendations

🎨 **Beautiful, Interactive Interface**

- Retro-vinyl themed UI with modern aesthetics
- Drag-and-drop file upload with visual feedback
- Real-time processing status and audio visualizations
- Smooth animations and responsive design

🔒 **Privacy-First Architecture**

- Human-made tracks: deleted immediately after analysis
- AI-generated tracks: temporarily processed for similarity matching only
- No long-term storage of user audio files
- Respects copyright and user privacy

## Tech Stack

### 🔧 Backend

- **Python & Flask**: RESTful API handling file uploads and predictions
- **PyTorch**: Custom CNN model (`AudioCNN`) for binary classification
- **Audio Processing**: `torchaudio` and `librosa` for Mel spectrogram conversion
- **Similarity Engine**: `torchopenl3` for deep audio embeddings + `faiss` for efficient vector search
- **Data Management**: `pandas` and `numpy` for metadata and embeddings

### 🎨 Frontend

- **React (Vite)**: Fast, modern framework with rapid development server
- **TypeScript**: Full type safety and better IDE support
- **Tailwind CSS**: Responsive, modern utility-first styling
- **shadcn/ui**: Accessible, high-quality component library
- **Custom Animations**: Vinyl turntable and waveform visualizations
- **Audio Playback**: Built-in audio player with visual feedback

## Installation & Setup

### Prerequisites

- **Node.js** v16+ & npm (for frontend)
- **Python** 3.10+ (for backend)
- **(Optional) CUDA 11.8+** for GPU acceleration (faster inference)
- **Git** for cloning the repository

### Step 1: Clone the Repository

```bash
git clone https://github.com/yourusername/auralof.git
cd auralof
```

### Step 2: Backend Setup

**Create and activate Python virtual environment:**

```bash
# Create virtual environment
python -m venv venv

# Activate (choose based on your OS)
# On macOS/Linux:
source venv/bin/activate

# On Windows (PowerShell):
.\venv\Scripts\Activate.ps1

# On Windows (Command Prompt):
venv\Scripts\activate
```

**Install Python dependencies:**

```bash
pip install -r requirements.txt
```

**Verify setup:**

```bash
python -c "import torch; print(f'PyTorch {torch.__version__}')"
python -c "import faiss; print('FAISS installed')"
```

> **Important**: Ensure model weights are present in `backend/models/audio_cnn_20epochs_best.pth` and embeddings in `backend/embeddings/`

### Step 3: Frontend Setup

**Install Node.js dependencies:**

```bash
npm install
```

**Verify Vite configuration:**

```bash
npm run vite --version
```

## Running the Application

### 🚀 Option 1: Run Both (Recommended)

```bash
npm run dev
```

This single command starts both:

- **Frontend**: Vite dev server at `http://localhost:5174/`
- **Backend**: Flask API at `http://localhost:8000/`

> Make sure you've activated the Python virtual environment before running this command

### 🖥️ Option 2: Run Separately

**Terminal 1 - Start Backend:**

```bash
# Activate virtual environment
# On Windows (PowerShell):
.\venv\Scripts\Activate.ps1

# On macOS/Linux:
source venv/bin/activate

# Start Flask server
python backend/app.py
```

✅ Backend runs at `http://localhost:8000/`

**Terminal 2 - Start Frontend:**

```bash
npm run dev:frontend
```

✅ Frontend available at `http://localhost:5174/`

### 🛠️ Development Workflow

1. Keep both servers running during development
2. Frontend auto-reloads on file changes (Vite HMR)
3. Backend requires manual restart on `.py` file changes
4. Open browser to `http://localhost:5174/` for the UI
